## 🛠 Сборка и установка

### Требования
- Java 11 JDK
- JOSM (последняя стабильная версия)

### Сборка из исходного кода
Чтобы собрать плагин независимо, перейдите в этот каталог и выполните:
```bash
export JAVA_HOME=/путь/к/java-11
./gradlew build
```
Сгенерированный JAR-файл будет находиться по адресу `build/libs/mdbuildings-v2-2.0.0-SNAPSHOT.jar`.

### Ручная установка
1. Скопируйте созданный `.jar` файл в каталог плагинов JOSM:
   - **Linux/macOS**: `~/.local/share/JOSM/plugins/`
   - **Windows**: `%APPDATA%\JOSM\plugins\`
2. Перезапустите JOSM.

## ⚡ Быстрый старт и пошаговый процесс

1. Откройте JOSM и загрузите область, которую вы хотите нанести на карту.
2. **Загрузите данные OSM** для этой области (`Ctrl+Shift+Down`).
3. Откройте **боковую панель MD-Buildings**, щелкнув значок Красного Зубра на левой панели.
4. Выберите желаемый **профиль** (например, `GEODATA (Roofs)` или `CADASTRU (Buildings)`).

### 🛠 Идеальный рабочий процесс картирования

Чтобы получить максимальную отдачу от системы двойного источника плагина, следуйте этому оптимальному пошаговому процессу для обычного сеанса картирования:

#### Шаг 1: Геометрия и высота (GEODATA)
1. На боковой панели выберите профиль **`GEODATA (Roofs)`**. Установите режим на **Полный импорт** (Full Import).
2. Найдите на спутниковом снимке неотрисованное здание.
3. Наведите курсор мыши внутрь контура здания и нажмите **Ctrl + Двойной щелчок** (2 левых клика мышью).
4. Плагин извлечет высокоточный контур крыши и автоматически рассчитает реальную высоту здания с помощью DTM API.

#### Шаг 2: Адресные данные (CADASTRU)
1. Теперь переключите профиль боковой панели на **`CADASTRU (Buildings)`**. 
2. Измените режим импорта на **Обновление тегов** (Tags Update) (это сохранит точную геометрию, которую вы только что импортировали на Шаге 1).
3. Наведите курсор мыши внутрь нового здания и нажмите **Ctrl + Двойной щелчок**.
4. Плагин запросит кадастровый реестр, очистит названия городов и улиц (удаляя префиксы типа mun./or. и т.д.), исправит любые ошибочные румынские диакритические знаки и добавит безупречные теги `addr:*` прямо к вашему зданию.

#### Шаг 3: Разрешение конфликтов и проверка
1. Если вы обновляете уже существующую область с устаревшими тегами, может появиться диалоговое окно конфликта тегов (Tag Conflict Dialog). Выберите, какие исходные данные наиболее точны для объединения.
2. Обращайте внимание на окно с предупреждением «Редкие теги» (Uncommon Tags), которое фиксирует некорректные устаревшие данные в OpenStreetMap.
3. Запустите валидатор JOSM, проверьте полученную геометрию и нажмите **Загрузить на сервер** (Upload)!

## 📖 Обзор функций

| Функция | Описание |
|---|---|
| **Прямой импорт WFS** | Получает контуры зданий в реальном времени с молдавских государственных геосерверов (GEODATA, CADASTRU). |
| **Интеллектуальное слияние** | Автоматически переводит номера домов CADASTRU и объединяет внутренние узлы адресов. |
| **Очистка адресов** | Нормализует румынские диакритические знаки (Ş/Ţ в Ș/Ț) и удаляет префиксы населенных пунктов (mun., or.) для идеальных значений тегов. |
| **Продвинутый расчет высоты** | Многоуровневое получение DTM с резервным вариантом (fallback) для расчета высот зданий, переключаемое в настройках. Автоматически очищает необработанные теги `z=*`, если DTM пропущен. |
| **Определение наложений** | Предотвращает дублирование импорта и управляет пространственными конфликтами с существующими данными OSM. |
| **3 режима импорта** | Полный импорт, обновление только геометрии или обновление только тегов. |
| **Разрешение конфликтов тегов** | Разрешает конфликты тегов между загруженными данными и существующими объектами OSM. |
| **Проверка жизненного цикла** | Предупреждает, если теги здания содержат префиксы жизненного цикла (например, `demolished:building`). |
| **Проверка редких тегов** | Выделяет нестандартные теги, которые могут потребовать ручной проверки. |

## 📖 Подробное руководство пользователя

### 1. Запуск импорта здания

- Удерживая **Ctrl**, **дважды щелкните** левой кнопкой мыши в любом месте карты.
- Плагин запросит на геосервере ближайшее здание в позиции курсора. 
- Если под курсором находится существующее здание OSM, плагин попытается его обновить в соответствии с активным режимом импорта.

---

### 2. Режимы импорта

Плагин поддерживает три различные стратегии импорта, настраиваемые на **боковой панели** или в **Настройках**:

| Режим | Что он делает |
|---|---|
| **Полный импорт** | Импортирует как геометрию (форму), так и все теги из исходных данных. Если поблизости есть подходящее здание, оно полностью заменяется (геометрия + теги). |
| **Обновление геометрии** | Обновляет только *форму* выбранного существующего здания, оставляя все существующие теги OSM нетронутыми. Идеально, когда контур здания изменился. |
| **Обновление тегов** | Обновляет только *теги* выбранного существующего здания, оставляя его геометрию нетронутой. Полезно, когда контур верен, но теги устарели. |

> [!TIP]
> Используйте **Полный импорт** при картировании новых зданий. Используйте **Обновление геометрии** или **Обновление тегов** при улучшении уже нанесенных на карту зданий.

---

### 3. Боковая панель (Toggle Dialog)

Щелкните значок **MD-Buildings** (логотип Красного Зубра) на левой панели JOSM, чтобы открыть боковую панель.

Боковая панель предоставляет:
- **Селектор профиля** — выберите профиль активного источника данных (например, `GEODATA (Roofs)`, `CADASTRU (Buildings)`).
- **Селектор режима импорта** — переключение между режимами «Полный», «Геометрия» и «Теги».
- **Отображение статуса** — показывает результат последнего импорта (`DONE`, `NO_DATA`, `CONNECTION_ERROR` и т. д.).
- **Быстрая статистика** — общее количество зданий, импортированных за этот сеанс.

---

### 4. Источники данных и профили

Плагин «из коробки» подключается к двум официальным молдавским геосерверам:

| Сервер | URL | Описание |
|---|---|---|
| **GEODATA** | `https://geodata.gov.md/geoserver/maps/wfs` | Национальный геопортал — предоставляет контуры зданий на уровне крыш |
| **CADASTRU** | `https://cadastru.md/geoserver/w_cbi/wfs` | Кадастровый реестр — предоставляет контуры зданий с номерами домов |

**Профили** определяют, какой слой WFS использовать и какие теги импортировать/исключать:

| Профиль | Сервер | Слой | Включенные теги | Примечания |
|---|---|---|---|---|
| **GEODATA (Roofs)** | GEODATA | `maps:lm17_area_roof_area` | Все, кроме `layer_name`, `layer`, `medium`, `source:building` | Лучше всего для геометрии крыш |
| **CADASTRU (Buildings)** | CADASTRU | `w_cbi:cad_cladiri` | Только `NRCASA` (преобразуется в `addr:housenumber`) | Лучше всего для адресных данных |

Вы можете добавлять, редактировать или удалять серверы и профили в **Настройки → MD-BUILDINGS → Data Sources**.

---

### 5. Профиль CADASTRU и обработка адресов

При использовании профиля **CADASTRU (Buildings)**:

- Плагин фильтрует все теги из ответа WFS, сопоставляя кадастровые поля с тегами OSM (например, `NRCASA` в `addr:housenumber`, `STREET` в `addr:street`, `CITY` в `addr:city`).
- **Очистка адресов**: Префиксы типа поселений (например, *mun., or., c., s.*) автоматически удаляются из названий городов, оставляя чистый формат названий населенных пунктов.
- **Нормализация диакритических знаков**: Устаревшие или некорректные румынские диакритические знаки (например, *Ş/ş, Ţ/ţ*) автоматически преобразуются в правильные символы с запятой внизу (*Ș/ș, Ț/ț*).
- Теги `building=yes` и `source=AGCC/Linemap2017` всегда добавляются автоматически.

---

### 6. Слияние внутренних узлов адресов

Эта функция автоматически находит **существующие узлы OSM**, расположенные *внутри* контура вновь импортированного здания, и **переносит их теги `addr:*`** непосредственно на здание.

**Включение этой функции:**
1. Откройте **Настройки** (`F12`) → **MD-BUILDINGS** → вкладка **Data Sources**.
2. Отметьте флажок: **"Merge interior address nodes into building"**.
3. Нажмите ОК и начните импорт.

> [!NOTE]
> Эта функция **отключена по умолчанию**. Включайте её только при работе в областях, где в OSM уже существуют адресные узлы и вы хотите объединить их с полигонами зданий.

---

### 7. Разрешение конфликтов тегов

Когда здание уже существует в OSM и плагин импортирует более новые данные с геосервера, могут возникнуть конфликты тегов. Плагин обрабатывает их автоматически по следующей логике:

- **Тег здания** (`building=*`): Существующее значение сохраняется, если оно более специфично, чем `yes` (например, `building=residential` сохраняется вместо перезаписи на `building=yes`).
- **Все остальные теги**: Новые теги с геосервера применяются поверх существующих. Существующие теги, отсутствующие в новых данных, как правило, сохраняются.

---

### 8. Продвинутый автоматический расчет высоты

Когда контур здания из источника **GEODATA** включает атрибут `z` (абсолютная высота крыши), плагин может автоматически рассчитать относительную высоту здания:

1. **Многоуровневый запрос DTM**: Запрашивает API ЦМР (Цифровой модели рельефа) с нескольких URL-адресов Geoserver (со встроенной логикой повторных попыток и резервных вариантов) для получения высоты земли в центральной точке здания.
2. Рассчитывает **относительную высоту** как: `height = z_roof - z_ground`.
3. Округляет результат до ближайших 0,1 метра.
4. Применяет тег `height=<значение>` к зданию.

> [!TIP]
> Эту функцию можно включить/выключить в **Настройках**. Если расчет DTM отключен или недоступен, необработанные теги `z=*`, полученные с Geoserver, автоматически удаляются, чтобы предотвратить загрязнение данных OSM значениями абсолютной высоты вместо относительной высоты здания.

---

### 9. Динамический радиус поиска BBOX

Плагин использует настраиваемый сдвиг `BBOX_OFFSET` для формирования точных запросов к областям WFS, позволяя пользователям корректно извлекать релевантные данные (например, адреса CADASTRU) вокруг активно выбранного здания или позиции курсора. Это предотвращает пропуски элементов во время запросов данных при более широкой топологии зданий.

## ⚙️ Настройки и конфигурация

Откройте Настройки с помощью `F12`, затем перейдите в раздел **MD-BUILDINGS**. В окне настроек есть следующие вкладки:

- **Вкладка Data Sources**: Управление подключениями к WFS геосерверам и профилями импорта.
- **Вкладка Stats**: Просмотр и сброс статистики импорта.
- **Вкладка Notifications**: Настройка отображаемых уведомлений и предупреждений.
- **Вкладка Uncommon Tags**: Управление списком «обычных» значений тегов зданий.
- **Вкладка Auto-remove Source Tags**: Управление значениями тега `source`, автоматически удаляемыми при обновлении.

## 🗄 Источники данных

### GEODATA — Национальный геопортал
- **URL**: `https://geodata.gov.md/geoserver/maps/wfs`  
- **Слой**: `maps:lm17_area_roof_area`  
- **Лучше всего для**: Точности геометрии, расчета высоты, общих форм зданий  

### CADASTRU — Кадастровый реестр  
- **URL**: `https://cadastru.md/geoserver/w_cbi/wfs`  
- **Слой**: `w_cbi:cad_cladiri`  
- **Лучше всего для**: Номеров домов (`addr:housenumber`), юридических границ зданий  

---

## 💡 Советы и рекомендации

1. **Используйте GEODATA для геометрии, CADASTRU для адресов.**
2. **Включите "Merge interior address nodes"** в городских районах с существующими адресными узлами.
3. **Всегда проверяйте индикатор статуса** на боковой панели после каждого импорта (`DONE`, `NO_DATA` и т. д.).
4. **Используйте режим «Обновление геометрии»**, когда контуры меняются, но теги должны быть сохранены.
5. **Внимательно изучайте диалог редких тегов.** Он отлавливает некорректные данные из источника.

## 🛠 Исправление проблем

### Сбой браузера справки JOSM
Если JOSM аварийно завершает работу при открытии меню справки на Java 16+, добавьте этот флаг в команду запуска JOSM:
```bash
--add-opens java.desktop/javax.swing.text.html=ALL-UNNAMED
```

## 🧹 Обслуживание репозитория

Чтобы репозиторий оставался компактным и удобным в обслуживании:
- Тестовые данные хранятся в `test/data/` и используются напрямую автоматизированными тестами.
- Устаревшие временные файлы и дубликаты тестов не должны храниться в отдельных папках верхнего уровня.
